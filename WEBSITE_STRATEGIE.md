## 1. Flexen App — Was muss auf die Website

### Kernfeatures (Hero-worthy)
Diese Features sind Flexens Unique Selling Points und muessen prominent praesentiert werden:

| Feature | Warum wichtig | Wie praesentieren |
|---------|--------------|-------------------|
| **KI-Coach "Julio"** | Groesster Differentiator — kein Wettbewerber hat einen vollintegrierten KI-Coach mit Function Calling | Live-Demo/Video: User spricht mit Julio, Julio erstellt Trainingsplan in Echtzeit |
| **KI-Essenserkennung** | Foto machen → sofort Kalorien/Makros — magischer Moment | Scroll-Animation: Handy fotografiert Essen → Ergebnis erscheint animiert |
| **Barcode-Scanner** | Instant Nutrition Lookup — alltagstauglich | Kurze Animation: Barcode wird gescannt → Naehrwerte poppen auf |
| **Ganzheitlicher Ansatz** | Ernaehrung + Training + Schlaf + Mental + Medikamente in EINER App | Bento-Grid das alle 5 Bereiche gleichzeitig zeigt |
| **Trainingsplan-Generator** | KI erstellt personalisierte Plaene mit Timer, Sets, Reps | Mockup des aktiven Workout-Screens mit Rest-Timer |
| **Schlaf-Tracking** | Health Connect Integration, automatisch | Schlaf-Kurven-Animation die sich aufbaut |
| **Journal mit Spracheingabe** | Voice-to-Text Tagebuch — einzigartig im Fitness-Bereich | Waveform-Animation die zu Text wird |
| **Freunde & Community** | Friend Codes, Nudges, Activity Sharing | Social-Proof-Element auf der Website |

### Alle App-Bereiche (fuer Feature-Seite)
1. **Home Dashboard** — Tagesueberblick mit Kalorien-Kreisen, Makros, Stimmung, Wasser
2. **Ernaehrung** — Mahlzeiten-Tracking, Rezepte, Kuehlschrank, Einkaufsliste, Medikamente
3. **Sport** — Trainingsplaene, Uebungsbibliothek, Schritte, verbrannte Kalorien
4. **Mind** — Schlaf-Tracking, Journal, Stimmungs-Tracking, Schreibimpulse
5. **KI-Chat** — Julio AI Assistant mit Function Calling
6. **Social** — Freunde, Nudges, Friend Codes
7. **Premium** — Erweiterte KI, unbegrenzte Analysen

### App-Branding-Assets (vorhanden)
- Logo: `Logo_Standard.png`, `Logo_White_Padded.png`
- Farben: Dark Blue `#001622` (Primary), Light-Varianten
- Font: Ubuntu (300, 400, 500, 700)

## 3. Anti-Generisch — Was Flexen einzigartig macht

### Das Problem: Warum die meisten Fitness-Websites gleich aussehen
- Template-Design (Squarespace/Webflow Templates)
- Stock-Fotos von laechelnden Menschen mit Hanteln
- Generische Headlines: "Your fitness journey starts here"
- Identische Sektionsreihenfolge: Hero → Features → Testimonials → Pricing → Footer
- Keine eigene visuelle Identitaet
- Austauschbare Farbpaletten (Blau/Gruen = "Gesundheit")

### Flexens Anti-Generisch-Strategie

#### 1. Eigene visuelle Sprache entwickeln
**Statt:** Stock-Fotos von Menschen im Gym
**Machen:** App-Screenshots in 3D-Device-Mockups mit Scroll-Animationen — die App IST das Produkt, nicht ein abstraktes Lifestyle-Versprechen

**Statt:** Generische Icons (Herz, Hantel, Apfel)
**Machen:** Custom Illustrations oder die eigenen SVG-Icons aus der App animiert

**Statt:** Helvetica/Inter/System Font
**Machen:** Distinktives Font-Pairing — z.B. **Druk Wide** (Headlines, Kraft/Energie) + **Inter** (Body, Klarheit) oder **Space Grotesk** + **DM Sans**

#### 2. Story statt Feature-Liste
**Statt:** "Track your calories, workouts, and sleep"
**Machen:** Scroll-Storytelling: Ein Tag im Leben eines Flexen-Nutzers

```
MORGEN: Julio weckt dich mit deinem Schlaf-Score
       → Scroll-Animation zeigt Schlaf-Kurve die sich aufbaut

FRUEHSTUECK: Foto vom Essen → KI erkennt sofort die Kalorien
       → Interaktive Demo: Nutzer sieht Live-Erkennung

MITTAG: Dein personalisierter Trainingsplan wartet
       → 3D-Phone-Mockup dreht sich, zeigt Workout-Screen

ABEND: Journal-Eintrag per Sprache
       → Waveform-Animation → Text erscheint

NACHT: Schlaf-Tracking startet automatisch
       → Bildschirm dimmt, Sterne-Animation
```

#### 3. Interaktive Elemente statt passive Praesentation
- **Live KI-Demo:** Nutzer kann auf der Website ein Foto hochladen und sieht eine Vorschau der Essenserkennung
- **Kalorienrechner:** Interaktives Tool direkt auf der Website
- **Quiz:** "Welcher Fitness-Typ bist du?" → Personalisierte Empfehlung
- **Scroll-controlled App Tour:** Phone-Mockup das sich beim Scrollen durch App-Screens bewegt

#### 4. Emotionale Trigger statt rationale Feature-Listen
- **Frustration ansprechen:** "Keine Lust mehr, jede Kalorie manuell einzutippen?"
- **Transformation zeigen:** Echte Nutzer-Stories mit konkreten Zahlen
- **Zeitersparnis quantifizieren:** "Spart durchschnittlich 12 Minuten pro Mahlzeit"
- **KI als Buddy, nicht als Tool:** "Julio kennt dich besser als dein Personal Trainer"

#### 5. Unerwartete Design-Entscheidungen
- **Horizontales Scrollen** fuer Feature-Showcase (wie Peloton Produkt-Karussell)
- **Cursor-Follower** der sich je nach Section aendert (Hantel-Icon bei Sport, Gabel bei Ernaehrung)
- **Sound-Design:** Optionale subtile Sounds bei Scroll-Milestones (wie App-Sounds)
- **Easter Eggs:** Versteckte Animationen die Nutzer entdecken koennen
- **Parallax mit Tiefe:** Verschiedene Ebenen bewegen sich unterschiedlich schnell

#### 6. Authentizitaet durch App-Integration
- Screenshots und Mockups direkt aus der echten App, nicht aus Figma-Designs
- Echte Nutzerdaten (anonymisiert) als Beispiele
- Die Website-Animationen spiegeln die App-Animationen wider (gleiche Easing-Kurven, gleiche Farben)
- QR-Code zum sofortigen App-Download prominent platziert

## 4. Design-System

### Farbpalette

#### Primary (aus App uebernommen)
```
--flexen-dark:       #001622    // Deep Navy — Haupt-Background
--flexen-dark-mid:   #0a2a3f    // Fuer Cards/Sections auf Dark
--flexen-dark-light:  #143d5a    // Hover-States, Borders
```

#### Akzentfarben (1 Hero-Akzent waehlen)
```
Option A — Electric Blue (wie WHOOP):
--flexen-accent:     #4A7DFF    // CTA-Buttons, Links, Highlights
--flexen-accent-glow: rgba(74, 125, 255, 0.3)  // Glow-Effekte

Option B — Energetisches Teal (wie Tonal):
--flexen-accent:     #44DDC4
--flexen-accent-glow: rgba(68, 221, 196, 0.3)

Option C — Warmes Orange (wie Strava/Freeletics):
--flexen-accent:     #FF6B35
--flexen-accent-glow: rgba(255, 107, 53, 0.3)
```

**Empfehlung:** Option A (Electric Blue) — differenziert von YAZIO (Gruen), Freeletics (Schwarz/Weiss), Strava (Orange) und signalisiert Tech/KI/Innovation.

#### Semantische Farben
```
--text-primary:      rgba(255, 255, 255, 0.95)   // Headlines
--text-secondary:    rgba(255, 255, 255, 0.70)   // Body Text
--text-tertiary:     rgba(255, 255, 255, 0.45)   // Captions, Labels
--text-quaternary:   rgba(255, 255, 255, 0.25)   // Disabled, Dividers

--success:           #34D399   // Gruene Akzente (Ernaehrung, Ziele erreicht)
--warning:           #FBBF24   // Warnungen
--error:             #EF4444   // Fehler
```

#### Gradient-System
```
--gradient-hero:     linear-gradient(135deg, #001622 0%, #0a2a3f 50%, #001622 100%)
--gradient-accent:   linear-gradient(135deg, #4A7DFF 0%, #7B61FF 100%)
--gradient-card:     linear-gradient(180deg, rgba(255,255,255,0.05) 0%, rgba(255,255,255,0.02) 100%)
--gradient-glow:     radial-gradient(circle at center, var(--flexen-accent-glow), transparent 70%)
```

### Typografie

#### Font-Pairing (Premium, nicht generisch)

**Headlines:** `Space Grotesk` (Gewichte: 500, 700)
- Geometrisch, modern, tech-forward
- Differenziert von Inter/Helvetica/Roboto die JEDER nutzt
- Erhaeltlich als Variable Font

**Body:** `DM Sans` (Gewichte: 400, 500, 700)
- Humanistisch, warm, gut lesbar
- Ergaenzt Space Grotesk perfekt
- Hervorragend auf allen Screengroessen

**Mono (Code/Daten):** `JetBrains Mono` oder `Geist Mono`
- Fuer Zahlen, Statistiken, technische Elemente
- Signalisiert Praezision und Tech-Kompetenz

#### Typografie-Scale (Fluid, wie Fitonist)
```css
--text-hero:    clamp(3rem, 5vw + 1rem, 7rem);      /* 48px → 112px */
--text-h1:      clamp(2.5rem, 4vw + 0.5rem, 5rem);   /* 40px → 80px */
--text-h2:      clamp(2rem, 3vw + 0.5rem, 3.5rem);    /* 32px → 56px */
--text-h3:      clamp(1.5rem, 2vw + 0.5rem, 2.5rem);  /* 24px → 40px */
--text-body:    clamp(1rem, 1vw + 0.5rem, 1.25rem);   /* 16px → 20px */
--text-small:   clamp(0.875rem, 0.8vw + 0.4rem, 1rem); /* 14px → 16px */
--text-caption: 0.75rem;                                /* 12px */
```

#### Typografie-Regeln
- `text-wrap: balance` auf Headlines (wie Linear)
- `font-feature-settings: "ss01", "ss02"` fuer Stylistic Alternates
- Zeilenhoehe: Headlines 1.1, Body 1.6, Captions 1.4
- Letter-Spacing: Headlines -0.02em (enger), Body 0
- Max. Zeilenbreite Body: 65-75 Zeichen (`max-width: 42rem`)

- Asymmetrische Layouts fuer visuelles Interesse (nicht alles zentriert)

## 5. Animations-Konzept

### Philosophie
> "Animationen muessen sich anfuehlen wie ein Gespraech, nicht wie ein Feuerwerk."
> — Zurueckhaltung schlaegt Uebertreibung. Linear nutzt 3200ms Timing. Apple animiert langsam und bewusst.

### Kern-Prinzipien
1. **Zweck vor Effekt** — Jede Animation muss Information vermitteln oder Orientierung geben
2. **Konsistentes Timing** — Gleiche Easing-Kurven ueberall: `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo)
3. **Performance-First** — Nur `transform` und `opacity` animieren (GPU-beschleunigt)
4. **Accessibility** — `@media (prefers-reduced-motion: reduce)` deaktiviert alle Animationen
5. **Mobile-Optimiert** — Weniger/einfachere Animationen auf Mobile

### Animation-Katalog

#### 1. Scroll-Storytelling (Hauptfeature der Website)
**Technik:** GSAP ScrollTrigger + Lenis Smooth Scrolling
**Konzept:** "Ein Tag mit Flexen" — User scrollt durch einen Tag

```
Scene 1: MORGEN (Scroll 0-20%)
  - Phone-Mockup faehrt von unten rein
  - Schlaf-Score-Kreis fuellt sich animiert
  - Hintergrund: Deep Navy → leichtes Dawn-Gradient

Scene 2: FRUEHSTUECK (Scroll 20-40%)
  - Phone dreht sich, zeigt Kamera-Screen
  - Essen-Foto erscheint, KI-Overlay animiert sich
  - Kalorien/Makros zaehlen hoch (CountUp Animation)

Scene 3: TRAINING (Scroll 40-60%)
  - Phone zeigt Trainingsplan
  - Uebungen scrollen durch
  - Timer-Animation, Reps-Counter

Scene 4: ABEND (Scroll 60-80%)
  - Journal-Screen erscheint
  - Waveform-Animation (Voice Input)
  - Text materialisiert sich Wort fuer Wort

Scene 5: NACHT (Scroll 80-100%)
  - Hintergrund wird dunkler
  - Schlaf-Tracking aktiviert sich
  - Sanfte Partikel-Animation (Sterne/Gluehwuermchen)
  - CTA: "Starte deinen ersten Tag mit Flexen"
```

#### 2. Micro-Interactions

**Buttons:**
- Hover: Sanfter Scale (1.02), Glow-Effekt hinter dem Button
- Magnetic Effect: Button "zieht" den Cursor leicht an (±10px Radius)
- Click: Kurzer Scale-Down (0.97) → Scale-Up (1.0), 200ms

**Cards:**
- Hover: Leichter Lift (translateY -4px), Shadow verstaerkt sich
- Tilt: Subtiler 3D-Tilt dem Cursor folgend (max ±5 Grad)
- Border: Gradient-Border die beim Hover erscheint

**Navigation:**
- Active State: Unterstrichen mit animiertem Indicator
- Mobile Menu: Slide-in von rechts mit staggered Item-Animation
- Scroll: Nav wird transparent → solid nach 100px Scroll

**Cursor (Desktop):**
- Custom Cursor: Kleiner Kreis (12px) der dem Mauszeiger folgt
- Ueber Links: Kreis vergroessert sich (32px), wird halbtransparent
- Ueber Bildern: Kreis wird zum "Explore" oder "View" Text
- Ueber CTAs: Kreis verschwindet, nativer Pointer

#### 3. Text-Animationen

**Headlines:**
- Reveal: Wort-fuer-Wort Erscheinen von unten, leicht gestaffelt (50ms pro Wort)
- Split: Buchstaben fliegen von verschiedenen Positionen zusammen
- Highlight: Akzentfarbe "malt" sich unter wichtigen Woertern

**Statistiken:**
- CountUp: Zahlen zaehlen von 0 hoch wenn sichtbar (z.B. "50.000+ Nutzer")
- Timing: 1.5-2s mit ease-out fuer natuerliches Abbremsen

**Body Text:**
- Fade-in-up: Absaetze erscheinen sanft beim Scrollen
- Stagger: 100ms Verzoegerung zwischen Absaetzen
- KEIN Buchstaben-Animation auf Body Text (zu ablenkend)

#### 4. Uebergangs-Animationen

**Page Transitions:**
- Clip-path Animation: Neue Seite "enthuellt" sich von der Mitte
- Oder: Smooth Fade mit leichtem Scale (0.98 → 1.0), 400ms

**Section Transitions:**
- Gradient-Shift: Hintergrundfarbe aendert sich smooth zwischen Sections
- Parallax-Divider: Dekorative Elemente die sich zwischen Sections bewegen

#### 5. 3D-Elemente

**Phone Mockup:**
- 3D-iPhone/Android-Mockup das sich beim Scrollen dreht
- Tool: Spline (einfach) oder Three.js/React Three Fiber (flexibel)
- Zeigt echte App-Screenshots die sich beim Scrollen wechseln
- Auf Mobile: Statisches Mockup (Performance)

**Optionale 3D-Akzente:**
- Schwebende geometrische Formen im Hintergrund (subtil, low-poly)
- Partikel-System fuer "KI"-Visualisierung
- Nur wenn Performance-Budget es erlaubt

#### 6. Lottie/Rive Animationen
Fuer komplexe, wiederverwendbare Animationen:

### Empfohlener Stack

|-----------|-----------|-------|
| **Framework** | **Astro 5** + React Islands | 40% weniger JS als Next.js, perfekte Core Web Vitals, trotzdem React fuer interaktive Teile |

| **Smooth Scroll** | Lenis | Butter-smooth Scrolling, perfekte GSAP-Integration |
| **3D** | Spline (einfach) / Three.js (komplex) | Phone-Mockup, optionale 3D-Effekte |
| **Micro-Animations** | Framer Motion | React-native Animationen, Layout-Animationen |
| **Complex Animations** | Rive oder Lottie | Julio-Avatar, Feature-Icons, Loading States |
| **Icons** | Lucide Icons | Konsistent, Tree-Shakeable, Open Source |
| **CMS (Blog)** | Astro Content Collections oder Contentlayer | Markdown-basiert, Git-gesteuert, kein externes CMS noetig |
| **Hosting** | Vercel oder Cloudflare Pages | Edge-Rendering, globales CDN, automatische Optimierung |
| **Analytics** | Plausible Analytics | DSGVO-konform OHNE Cookie-Banner, EU-gehostet |
| **Forms** | Formspark oder Formspree | Kontaktformular ohne Backend |
| **Email** | Resend oder Loops | Transaktionale Emails + Newsletter |
| **A/B Testing** | PostHog (self-hosted) | DSGVO-konform, Feature Flags, Heatmaps |
| **Image CDN** | Cloudinary oder Vercel Image Optimization | WebP/AVIF, responsive, lazy-loaded |

### Warum Astro statt Next.js fuer die Marketing-Website

1. **Performance:** Astro liefert 0 KB JavaScript by default — nur wo interaktiv noetig
2. **SEO:** Statisches HTML = perfektes Crawling, perfekte Core Web Vitals
3. **Flexibilitaet:** React-Komponenten wo noetig (interaktive Demos, 3D), statisch wo nicht
4. **Einfachheit:** Weniger Komplexitaet als Next.js App Router
5. **Build-Speed:** Schnellere Builds als Next.js

## 11. Rechtliche Anforderungen (Deutschland/EU)

### PFLICHT — Ohne diese Elemente NICHT live gehen

#### 1. Impressum
- Vollstaendiger Firmenname, Adresse (physisch, keine Postfach), Email, Telefon
- Fuer GmbH: Registergericht, Registernummer, Geschaeftsfuehrer, USt-IdNr.

#### 2. Datenschutzerklaerung
- Verantwortlicher, DSB (falls zutreffend), Rechtsgrundlagen, Datenkategorien
- Aufbewahrungsfristen, Drittanbieter-Empfaenger, Betroffenenrechte
- Muss Website-Analytics, Kontaktformulare, Newsletter, App-Datenverarbeitung abdecken
- Sprache: Muss in der Seitensprache sein (Deutsch fuer DE)
- Empfohlener Generator: eRecht24 (kostenpflichtig, aber rechtssicher)

#### 3. Cookie Consent Banner
- MUSS vor ALLEN nicht-essentiellen Cookies/Tracking erscheinen
- MUSS bieten: "Alle akzeptieren", "Alle ablehnen", granulare Auswahl
- "Alle ablehnen" MUSS gleich prominent sein wie "Alle akzeptieren"
- KEINE Dark Patterns (keine vorausgecheckten Boxen)
- Muss alle nicht-essentielle Scripts bis zur Einwilligung blockieren
- Re-Consent nach max. 12 Monaten
- **Empfohlene CMPs:** Cookiebot oder Usercentrics (beides deutsche Firmen)
- **Pro-Tipp:** Plausible Analytics braucht KEINEN Cookie-Consent (cookieless)

#### 4. TTDSG-Compliance
- Gilt fuer ALLE Cookies und aehnliche Technologien
- Consent erforderlich fuer: Analytics-Cookies, Marketing-Cookies, Personalisierung
- Kein Consent noetig fuer: Strikt notwendige Cookies (Session, Sprache, Consent-Status)

#### 5. Weitere Pflichten
- **AGB** falls Premium-Abo ueber Website verkauft wird
- **Widerrufsbelehrung** bei Verbraucher-Verkaeufen
- **Preisangabenverordnung:** Alle Preise inkl. MwSt., als "inkl. MwSt." gekennzeichnet
- **BFSG (ab Juni 2025):** Barrierefreiheits-Anforderungen (WCAG 2.1 AA) fuer digitale Produkte

#### 6. Bussgeld-Risiken
- DSGVO: Bis zu 20 Mio EUR oder 4% des globalen Jahresumsatzes
- TTDSG: Bis zu 300.000 EUR
- Impressumspflicht-Verstoss: Bis zu 50.000 EUR

## 12. Performance-Budgets

### Seitengewicht-Budgets
| Resource | Budget (komprimiert) |
|----------|---------------------|
| **Gesamt-Seitengewicht** | < 500 KB |
| JavaScript | < 150 KB |
| CSS | < 50 KB |
| Bilder (Above Fold) | < 200 KB total |
| Fonts | < 100 KB (max 2 Font-Familien, 2-3 Weights) |
| Time to Interactive | < 3.5s auf 3G |

### Bild-Optimierung
- Format: WebP primaer, AVIF fuer unterstuetzende Browser, PNG/JPG Fallback
- Responsive: `srcset` + `sizes` oder Astro `<Image>` Komponente
- Lazy Loading: Alles unter dem Fold mit `loading="lazy"`
- Hero-Bild: Preloaded, NICHT lazy-loaded
- Max. Dimensionen: In Display-Groesse ausliefern
- Kompression: 80% Qualitaet fuer Fotos, Lossless fuer UI

### Font-Optimierung
- `font-display: swap` mit preloaded WOFF2
- Nur benoetigte Weights laden (nicht alle 9)
- Variable Fonts bevorzugen (1 File statt 4)

### Animation-Performance
- NUR `transform` und `opacity` animieren (GPU-beschleunigt)
- `will-change: transform` sparsam einsetzen
- Intersection Observer fuer Scroll-Trigger (nicht Scroll Event Listener)
- `@media (prefers-reduced-motion: reduce)` — alle Animationen deaktivieren
- Device-spezifische Qualitaets-Stufen (weniger 3D auf Mobile)

### Caching-Strategie
```
Statische Assets (Images, Fonts, CSS, JS):
  Cache-Control: public, max-age=31536000, immutable

HTML-Seiten:
  Cache-Control: public, max-age=0, s-maxage=3600, stale-while-revalidate=86400

API-Responses:
  Cache-Control: private, no-cache
```
